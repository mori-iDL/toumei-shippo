## 🎯 最優先：P0課題の解決

**P0 = パラメータを変えても見た目が十分に変わらない**

異なる赤ちゃんのデータで生成した時、「一目で違う」と分かるパターンにすること。

### 着手順序の提案

1. **現状の可視化** — 5人分の異なるテストデータでパターンを並べ、差異の程度を把握
2. **感度分析** — どのパラメータが視覚的差異に最も影響するか特定（design-tunerの出番）
3. **チューニング** — 感度の低いパラメータの振幅・スケール・密度を拡大
4. **検証** — 再度5パターン比較で改善を確認（qa-testerの出番）

### 美意識の制約（姉が見ている視点）

- 「違う」だけでなく**「美しく違う」**こと
- 各樹種の木目特性との調和を壊さない
- 阿部さんの哲学「未完成がいい」「100人いれば100人の完成形」との整合
- **決定論的であること（同じ入力 = 必ず同じ出力）は絶対に守る** → validate-htmlフックが自動チェック済み

### Git運用の提案

P0着手前にブランチを切ることを推奨：
```bash
git checkout -b feature/p0-visual-diversity
```

## 🔧 技術リマインダー

### 三層翻訳アーキテクチャ

- **Layer 1（入力）**: 名前（母音分析）、誕生日（季節）、出生時刻（ブレンド比率）、体重・身長、出生地、天気
- **Layer 2（変換）**: 季節→主アルゴリズム（春=フィボナッチ、夏=ボロノイ、秋=反応拡散、冬=フラクタル）、母音→副アルゴリズム（あ=フロー、い=正弦波、う=パーリン、え=リサージュ、お=波動）、時刻→ブレンド比率
- **Layer 3（出力）**: 9樹種プリセット × デュアルモード（レーザー/V溝NC）

### 過去のデバッグ教訓

- `<select>`は`input`イベント不発 → `change`で分離
- Flow Fieldの関数シグネチャ → `fn.length`で動的検出
- 反応拡散: 800→400イテレーション
- 50msデバウンス
