## デザインコード

製品仕様管理書（旧称「てもり」時代）に9デザインコードが定義。
パラメトリックデザインへの転換後、赤ちゃんデータから無限のバリエーションが生成可能に。

## 三層翻訳アーキテクチャ

### Layer 1: 入力パラメータ
- 名前（ひらがな）→ 母音分析 → 副アルゴリズム
- 誕生日 → 季節判定 → 主アルゴリズム（春:フィボナッチ、夏:ボロノイ、秋:反応拡散、冬:フラクタル）
- 出生時刻 → ブレンド比率
- 体重 → 力強さ/繊細さ（質的影響）
- 身長 → 伸びやかさ/凝縮（質的影響）
- 天気 → 乱流度

### Layer 2: 自然計算アルゴリズム
主アルゴリズム（4季節） × 副アルゴリズム（5母音:あいうえお） × 各パラメータの質的影響

### Layer 3: 木の固有特性
樹種ごとのlineSpacing / lineWeight / grainStrength設定

## 二重出力モード

| モード | 用途 | 出力形式 | 特徴 |
|--------|------|----------|------|
| レーザー | 量産、繊細な模様 | SVG, PNG | 精密ベクトルライン、均一線幅 |
| V溝NC | 一品物、彫刻的表現 | SVG, DXF | 可変幅・可変深さ、立体的 |

## 加工実測データ

- **xTool P2S**: 5分2秒/枚, Power100%, Speed400mm/s, 2回
- **NC加工**（旧30mm厚時代）: 10周切り出し、下2mm残す、12個/35-45分
- **レーザー加工費**: ¥480/枚（4分 × ¥120/分）

## 安全基準

製品仕様管理書に安全基準記載あり（STマーク基準準拠を目指す）

## SVG/DXF出力の加工互換性

- SVGのopacity属性が加工ソフトで色を変質させる → 純色化で対応済み
- 副アルゴリズムの繊細な線（0.2mm幅）はNC加工には細すぎる → 閾値フィルタで対応済み
- muted colorは人の目には美しいがソフトウェアの色識別には純色が確実
