### 技術的な要点

**stop-notion-sync.sh v2の変更:**
- `PATCH /v1/blocks/{page_id}/children`（ページ末尾追記）→ `POST /v1/pages`（DB新規ページ作成）
- トークン取得優先順位を修正: `.claude.json` > `$NOTION_API_KEY` > `.zshrc`
  - 旧: `$NOTION_TOKEN` 環境変数を最優先 → MCP OAuthトークンを誤取得する可能性があった
- プロジェクト名をDB selectオプションにマッピング（Toumei→とうめいしっぽ、等）
- 経過時間を「45分」→45、「1時間23分」→83 のように数値変換

**Notion ID二重性（重要な学び）:**
- MCP data_source_id ≠ API database_id。同じDBでもアクセス経路でIDが違う
- セッション作業ログDB: MCP `444646a6...` / API `ed9aa864...`
- 📮申し送りDB: MCP `c444b65a...` / API `6c7479c3...`
- curlでMCP IDを使うと404。逆もまた然り
