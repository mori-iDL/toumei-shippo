# 視覚的評価基準 — Visual Evaluation Criteria

## 「十分に違う」の定義

### 人間の知覚基準
70mm×70mmの木のパネルを並べたとき:
1. **1m離れて見て**: 「違うパターンだ」と即座にわかる
2. **手に取って比べて**: 「どこがどう違うか」を説明できる
3. **同じ季節の子同士でも**: 名前・時刻・体重の違いが模様に表れる

### 定量的基準（SSIM: Structural Similarity Index）
- **異なるテストケース間**: SSIM < 0.70（十分に異なる）
- **同じ季節・異なる名前**: SSIM < 0.85（名前の影響が見える）
- **同一データ再実行**: SSIM = 1.00（完全に再現可能）
- **レーザーvsV溝NC**: SSIM < 0.60（モードの違いが明確）

## 評価対象の軸

### 1. 構造的差異（Structural）
- パターンの全体的な形状が異なるか
- 線の密度・方向が変わるか
- 空白領域の配置が異なるか

### 2. テクスチャ差異（Textural）
- 線の太さ・間隔のリズムが変わるか
- 細部の複雑さが異なるか
- 粗密のグラデーションが変わるか

### 3. 空間的差異（Spatial）
- パターンの重心が移動するか
- 放射・求心の方向性が変わるか
- エッジ（端）での処理が異なるか

## 自動評価パイプライン

### Puppeteer/Playwrightスクリプト
```
1. HTMLをヘッドレスブラウザで開く
2. テストケースのパラメータをフォームに入力
3. Canvas描画完了を待機（requestAnimationFrame + setTimeout 500ms）
4. レーザーキャンバスのスクリーンショットを保存
5. V溝NCキャンバスのスクリーンショットを保存
6. 次のテストケースへ
```

### 画像比較（Node.js: pixelmatch / ssim.js）
```
1. 2枚の画像をピクセル単位で比較
2. 差分画像を生成（赤=差異あり）
3. SSIM値を算出
4. 閾値判定
```

### 評価レポート生成
```
1. 全ペアのSSIM行列を生成
2. 閾値を下回るペアをハイライト
3. 差分画像をtmp/screenshots/に保存
4. Markdown形式で報告
```

## Digital Nature美的チェックリスト

自動評価では測れない美的品質。Zineの最終判断が必要な項目:

### 必須（Must）
- [ ] 「機械が作った」とは感じない有機性がある
- [ ] 70mmの中で模様として成立している（疎すぎず密すぎず）
- [ ] レーザー加工で物理的に再現可能な線幅（0.1mm以上）
- [ ] 木目との相性を想像できる

### 望ましい（Should）
- [ ] パターンを見て「何かの自然現象」を連想できる
- [ ] 季節ごとの性格の違いが直感的にわかる
- [ ] 時間が経っても飽きないと感じる
- [ ] 赤ちゃんの物語（名前・季節・天気）との接続を感じられる

### あれば素晴らしい（Nice to have）
- [ ] 裏返しても美しい（パターンの対称性）
- [ ] 複数枚並べたときにリズムが生まれる
- [ ] 光の当たり方で表情が変わりそうなテクスチャ

## パラメータ空間の探索指針

差異が不十分な場合の調整優先順位:

1. **主アルゴリズムの基本構造を大胆に変える**
   - フィボナッチの巻き数、ボロノイの点数、分岐の深さ等
   - 微調整ではなく、構造レベルの変化を狙う

2. **副アルゴリズムの影響度を上げる**
   - 現在の主副比率を見直し、副の存在感を強化
   - 混合方法を変更（加算→乗算、マスク、位相シフト等）

3. **天気パラメータの乱流度を拡大**
   - 晴れ→嵐のような極端な差を許容
   - ノイズの周波数・振幅のレンジを広げる

4. **体重・身長の影響を視覚的に明確化**
   - 密度だけでなく、線の太さや曲率にも反映
