# 視覚的評価 — Visual Evaluation

## 動的コンテキスト

### 安全テスト基盤（src/toumei_shippo_designer.html L2826-2960）
- `runSafetyTest()`: 144パターン（V溝108 + レーザー36）の暗度計測
- `measureDarkness(canvas)`: brightness < 100 のピクセル割合(%)を算出
- 閾値: WARN 55% / FAIL 65%
- テスト名: 「ゆいか」（3母音 u,i,a でalpha累積をストレステスト）

### パラメータ正規化（P0解決の実装知識）
- `normalizeWeight(w)`: 体重を -1〜+1 に正規化（中心: 3000g）
- `normalizeHeight(h)`: 身長を -1〜+1 に正規化（中心: 49cm）
- 体重の質的影響: 線の太さ、要素サイズ、構造の粒度（力強さ ↔ 繊細さ）
- 身長の質的影響: 構図の凝縮/展開、ストロークの長さ（凝縮 ↔ 伸びやか）

### DRY化共通関数（L1012-1025）
- `vgrooveScale(params, mode)`: V溝の線幅スケール計算
- `clampLW(value)`: 線幅を最大10pxにキャップ（画面真っ黒防止）
- `vgrooveSecondaryAlpha(alpha)`: 副アルゴリズムの透明度を抑制（0.15以上）

## Step 1: 評価目的の確認

| 目的 | 何を測るか | 使う指標 |
|------|-----------|---------|
| **差異測定** | 異なるパラメータで十分に違うパターンが出るか | SSIM + 3軸定性評価 |
| **回帰テスト** | コード変更で意図しない視覚的破壊がないか | SSIM ≒ 1.0（同一入力）+ 暗度変化 |
| **チューニング** | アルゴリズム調整の効果を定量評価 | SSIM変化量 + 暗度変化量 |
| **美的評価** | Digital Nature哲学に沿っているか | チェックリスト（Zineの最終判断） |

## Step 2: テストケース選定

### 標準テストペア

| ペア | 条件A | 条件B | 期待SSIM | 測定対象 |
|------|-------|-------|----------|---------|
| 季節差 | 春・はるか | 冬・めい | < 0.50 | 主アルゴリズム差 |
| 名前差 | 春・はるか | 春・りく | < 0.70 | 副アルゴリズム差（母音: a,u,a vs i,u） |
| 時刻差 | 正午 | 真夜中 | < 0.80 | 主副ブレンド比差 |
| 体重差 | 2000g | 5000g（同条件） | < 0.75 | 力強さ/繊細さの質的差 |
| 身長差 | 40cm | 60cm（同条件） | < 0.75 | 凝縮/展開の質的差 |
| モード差 | レーザー | V溝NC | < 0.60 | 出力モードの視覚差 |
| 樹種差 | スギ | ケヤキ | < 0.85 | lineWeight/lineSpacing差 |
| 天気差 | 晴れ | 雨 | < 0.85 | 乱流度の差 |
| 同一再実行 | はるか(1) | はるか(2) | = 1.00 | 決定論的描画の証明 |

### 境界値テスト

体重・身長の質的変化が十分に表れるかの検証:

| テスト | 値 | 確認ポイント |
|--------|---|-------------|
| 最軽量 | 2000g × 40cm | 最も繊細で凝縮されたパターン |
| 中央値 | 3000g × 49cm | ニュートラルなパターン |
| 最重量 | 5000g × 60cm | 最も力強く伸びやかなパターン |
| 軽+長 | 2000g × 60cm | 繊細だが伸びやかな矛盾感 |
| 重+短 | 5000g × 40cm | 力強いが凝縮された矛盾感 |

## Step 3: 定量評価

### 3.1 暗度計測（既存安全テストの活用）

`?debug=1`で起動する144パターン安全テストが**基底レイヤー**。
暗度は「破壊的に悪い」の検出 = 必要最低条件。

| 指標 | 合格基準 | 意味 |
|------|---------|------|
| 暗度 < 55% | OK | 加工可能な密度 |
| 55% ≤ 暗度 < 65% | WARN | 密すぎるが許容範囲 |
| 暗度 ≥ 65% | FAIL | 画面が真っ黒 → 加工不可 |

### 3.2 SSIM（構造類似性指数）

SSIMは「十分に違う」の検出 = 品質の積極的証明。暗度テストとは補完関係。

| カテゴリ | SSIM基準 | 意味 |
|---------|---------|------|
| 異なるテストケース間 | < 0.70 | 十分に異なる |
| 同じ季節・異なる名前 | < 0.85 | 名前の影響が見える |
| 同一データ再実行 | = 1.00 | 完全に再現可能 |
| レーザー vs V溝NC | < 0.60 | モードの違いが明確 |

### 3.3 3軸定性評価

自動化が難しい評価。目視またはdesign-tunerエージェントで実施:

**構造的差異（Structural）**
- パターンの全体的な形状が異なるか
- 線の密度・方向が変わるか
- 空白領域の配置が異なるか

**テクスチャ差異（Textural）**
- 線の太さ・間隔のリズムが変わるか
- 細部の複雑さが異なるか
- 粗密のグラデーションが変わるか

**空間的差異（Spatial）**
- パターンの重心が移動するか
- 放射・求心の方向性が変わるか
- エッジ（端）での処理が異なるか

## Step 4: 美的評価 — Digital Natureチェックリスト

自動評価では測れない美的品質。**Zineの最終判断が必要**な項目。

### 必須（Must）
- [ ] 「機械が作った」とは感じない有機性がある
- [ ] 70mmの中で模様として成立している（疎すぎず密すぎず）
- [ ] レーザー加工で物理的に再現可能な線幅（0.1mm以上）
- [ ] 木目との相性を想像できる

### 望ましい（Should）
- [ ] パターンを見て「何かの自然現象」を連想できる
- [ ] 季節ごとの性格の違いが直感的にわかる
- [ ] 時間が経っても飽きないと感じる
- [ ] 赤ちゃんの物語（名前・季節・天気）との接続を感じられる

### あれば素晴らしい（Nice to have）
- [ ] 裏返しても美しい（パターンの対称性）
- [ ] 複数枚並べたときにリズムが生まれる
- [ ] 光の当たり方で表情が変わりそうなテクスチャ

## Step 5: パラメータ空間の探索指針

差異が不十分（SSIM基準を満たさない）場合の調整優先順位:

### 1. 主アルゴリズムの基本構造を大胆に変える
- フィボナッチの巻き数、ボロノイの点数、分岐の深さ等
- 微調整ではなく、構造レベルの変化を狙う
- 体重による`normalizeWeight(w)`の影響度を確認（-1〜+1の全域で構造変化するか）

### 2. 副アルゴリズムの影響度を上げる
- 現在の主副比率を見直し、副の存在感を強化
- 混合方法を変更（加算→乗算、マスク、位相シフト等）
- `vgrooveSecondaryAlpha()`の下限0.15は適切か確認

### 3. 天気パラメータの乱流度を拡大
- 晴れ→嵐のような極端な差を許容
- ノイズの周波数・振幅のレンジを広げる

### 4. 体重・身長の影響を視覚的に明確化
- `clampLW()`の最大10pxキャップ内で質的変化が十分か
- 密度だけでなく、線の太さや曲率にも反映
- 「軽い子＝繊細、重い子＝力強い」が一目でわかる程度に

## 品質ゲート

| 軸 | ◎ 合格 | ○ 条件付き | △ 要改善 |
|----|--------|-----------|---------|
| 暗度 | 144パターン全OK(< 55%) | WARN数件(55-65%) | FAIL 1件以上(≥ 65%) |
| SSIM差異 | 全テストペアが基準値クリア | 1-2ペアが基準±0.05 | 3ペア以上が基準未達 |
| 決定論性 | 同一入力再実行 SSIM=1.00 | — | SSIM < 1.00（非決定的描画） |
| Digital Nature | Must全項目チェック済み | Should 半数以上 | Must未達項目あり |
| 木との相性 | 9樹種で木目との調和を想像可能 | 5-8樹種で想像可能 | 4樹種以下 |
